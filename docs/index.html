System Architecture

Three dedicated nodes, each optimized for its role. Communicating over WiFi and Serial.

Node 1 handles real-time ML (latency-critical), Node 2 handles UI and safety (compute-heavy), Node 3 handles motors (reliability-critical).

Node 1: Coral Dev Board Mini

ARM CPU + Edge TPU + ADS1299 ADC
  • 8-channel 24-bit EEG acquisition via SPI
  • 50Hz notch + 8-30Hz bandpass filtering
  • EEGNet inference on Edge TPU
  • Online adaptation (pseudo-label learning)
  • Calibration recording + auto fine-tune

Node 2: Raspberry Pi 5

4GB RAM + WiFi + USB Serial
  • NiceGUI dark-theme web dashboard
  • Drive / Calibrate / Settings tabs
  • Safety logic + heartbeat timeout
  • Serial relay to Arduino
  • Visual calibration cue protocol

Node 3: Arduino Mega 2560

ATmega2560 + L298N H-Bridge
  • 6-state finite state machine
  • Smooth PWM ramp engine
  • Hardware E-stop interrupt (ISR)
  • Battery voltage monitor + cutoff
  • Serial watchdog (2s timeout)

Signal Pipeline

From brain to wheelchair in under 10 milliseconds.

ADS1299 (250 SPS, 8ch, 24-bit) --> 50Hz Notch (IIR, Q=30) --> 8-30Hz Bandpass (Butterworth, order 4) --> 1s Window (250 samples) --> EEGNet (1,684 params, int8) --> Confidence Gate (40%) --> Vote Smoother (window=3) --> UDP TX --> Safety --> Motor PWM
Frequency response of the DSP chain. The notch filter removes 50Hz mains noise, the bandpass isolates mu (8-13Hz) and beta (13-30Hz) motor imagery rhythms.
Simulated right hand motor imagery. C3 (left motor cortex) shows clear mu/beta desynchronization during the imagery period. The classifier outputs "R" (steer right) with high confidence.

EEGNet Model

Compact CNN designed for EEG. 1,684 parameters -- small enough for Edge TPU int8 quantization.

During personal calibration, Block 1 and 2 are frozen (temporal and spatial features learned from 8 subjects). Only Block 3 and the classifier head are fine-tuned on personal data.

Results

Training and evaluation on BCI Competition IV-2a dataset (9 subjects, 4-class motor imagery).

Cross-subject pre-training on subjects 1-8 with PyTorch + CUDA. Best validation accuracy: 43% (chance level: 25%). Early stopping at epoch ~120.
Cross-subject confusion matrix. Forward and Right are best classified. Left/Stop confusion is expected -- both involve subtle imagery patterns.
Per-class accuracy before and after personal calibration (SITL, subject 9). All classes improve. Real EEG calibration is expected to reach 75-85%.
Per-component latency profile. EEGNet inference dominates at 4ms. Total pipeline uses 7.9ms of the 500ms budget -- 98.4% headroom.

Safety System

Three independent layers. Any single layer can stop the wheelchair.

LayerLocationMechanisms
Software Node 1 + 2 Confidence threshold (40%), majority vote (window=3), heartbeat timeout (3s), speed limiter
Firmware Node 3 (Arduino) Serial watchdog (2s), battery cutoff (10.5V), PWM ramp limiting, state validation
Hardware Electrical Physical E-stop button wired directly to motor driver enable -- bypasses all software

Quick Start

All software runs on a standard development PC with an NVIDIA GPU.

$ git clone https://github.com/Bumply/bitirme.git NeuroDrive
$ cd NeuroDrive
$ pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
$ pip install moabb mne numpy scipy scikit-learn matplotlib onnx nicegui pyserial

$ python node1_sitl_pipeline.py   # test DSP with replayed EEG
$ python node1_training.py      # train EEGNet (needs GPU)
$ python node1_calibrate.py     # personal fine-tuning
$ python node1_inference.py     # real-time SITL inference
$ python node2_dashboard.py     # dashboard at localhost:8080
$ python bench_test.py         # run all tests
